# 学术研究智能助手

> 面向社科研究的 AI 辅助分析工具

## 为什么做这个

定性研究中，访谈编码是最耗时也最机械的环节——二十份访谈记录，逐行标注概念、归类主题、提炼发现，重复劳动占比超过 60%。真正需要研究者判断的「这个编码对不对」「主题之间什么关系」，反而被疲劳拖慢了。

这个工具的目标不是替代研究者，而是把机械部分交给 AI，让人专注在需要判断力的地方。

## 设计理念

### 人机协作，而非人机替代

工具输出的是**编码初稿**，研究者在此基础上筛选、修正、深化。这和 NVivo 等 CAQDAS 工具的定位一致——辅助而非替代。

### 编码逻辑参考扎根理论

Prompt 模板参照 Strauss & Corbin 的开放编码流程设计：
1. 逐段提取概念（开放编码）
2. 概念归类为范畴（主轴编码）
3. 范畴间建立联系（选择性编码）

实际使用中简化了第三步，侧重前两步的效率提升。

### 结构化输出，可追溯可复核

每个编码都标注原始引文，方便研究者回溯检验。生成的结果不是黑箱结论，而是可审查的研究素材。

## 功能

| 功能 | 说明 | 输入 → 输出 |
|------|------|-------------|
| 智能编码 | 逐段识别关键概念，生成方括号编码 | 访谈文字 → 编码+主题+发现 |
| 横向对比 | 多份访谈的结构化对比 | 多份访谈 → 对比矩阵 |
| 发现提炼 | 生成学术规范的描述性段落 | 编码结果 → 研究发现初稿 |

### 示例

输入：
> "我会把它们抱在怀里，感觉心情就会好很多"

输出：
```
编码：【情感功能：情绪调节】【陪伴缓解焦虑】
主题：【情感慰藉】
发现：肢体接触可触发即时情绪改善
引文："我会把它们抱在怀里，感觉心情就会好很多"
```

## 技术实现

基于 OpenClaw 框架开发，核心是一个多步骤的结构化 Prompt 模板：

```
访谈文本 → 逐段概念提取 → 编码归类 → 主题聚类 → 发现提炼 → 结构化输出
```

关键设计：
- **Prompt 链式拆分**：编码、聚类、提炼分步执行，避免单次输出丢失细节
- **引用追踪**：每个编码强制标注原文出处，保证可复核性
- **编码规范**：方括号命名法 + 主题层级结构，与学术写作习惯对齐

## 适用场景

- 社会科学定性研究
- 深度访谈 / 半结构化访谈
- 焦点小组记录分析
- 扎根理论研究的编码阶段

## 已知局限

- 语境复杂的段落可能需要人工调整编码
- 跨访谈对比功能尚在完善中
- 当前仅支持中文访谈文本
- 不处理定量数据（CGSS 等）
